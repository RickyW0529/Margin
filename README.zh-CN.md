# Margin

Margin 是一个本地运行的 A 股研究助手。它把数据采集、量化筛选、新闻/公告证据、AI 复核和 Dashboard 推荐串成一条可追溯流程。

它不会自动交易，不管理券商账户，不承诺收益，也不构成投资建议。

## 1. 这个项目是干嘛的？

Margin 每天做一件事：把“哪些股票值得继续研究”变成一份有证据、有风险说明、能回看的研究结果。

```text
数据源 -> 质量检查 -> 量化筛选 -> 证据/RAG -> Agent 复核 -> Dashboard
```

当系统推荐一只股票时，你应该能看到：

- 为什么它被选中；
- 使用了哪些行情、财务、新闻或公告数据；
- AI 是否降低仓位、删除候选，或保留候选；
- 这次结论对应的时间点和证据链。

## 2. 我为什么要使用这个项目，能为我带来什么？

Margin 的价值不是只给你一个股票代码，而是给你一条研究链路。

| 需求 | Margin 提供什么 |
| --- | --- |
| 每天有候选股票 | 自动刷新候选池并展示在 Dashboard。 |
| 想知道推荐理由 | 展示量化评分、风险、证据和 AI 复核结论。 |
| 不想让 AI 胡说 | Agent 只能读已入库、可追溯的数据和证据。 |
| 想控制风险 | AI 复核可以降低权重或删除量化候选。 |
| 想复盘历史 | 推荐结果、证据、时间点和 Agent 产物会保存。 |
| 想本地掌控 | 数据库、密钥和任务都在本地环境里运行。 |

## 4. 我要怎么使用这个项目？

启动本地应用：

```bash
cp .env.example .env
python scripts/dev.py restart
```

打开：

```text
http://localhost:3000
```

推荐流程：

1. 在设置页配置数据源和模型 Provider。
2. 在 Dashboard 启动今日研究。
3. 查看推荐股票、评分、风险和证据。
4. 在首页追问“为什么推荐”“主要风险是什么”。
5. 把系统输出当作研究清单，再做自己的判断。

开发验证：

```bash
pip install -e ".[dev,data]"
ruff check src tests
pytest -q
```
