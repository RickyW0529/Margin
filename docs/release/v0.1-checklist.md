# Margin v0.1 Release Checklist

This checklist is for the initial public v0.1 tag. It focuses on local
reproducibility, deterministic tests, CI reliability, and clear product
boundaries.

## Required Local Startup Path

```bash
cp .env.example .env
python scripts/dev.py restart
```

Expected services:

- PostgreSQL starts through Docker Compose when it is not already healthy.
- Alembic migrations run before application processes start.
- Versioned config bootstrap runs before application processes start.
- API listens on `http://127.0.0.1:8000`.
- Worker process starts in the same checkout.
- Web app listens on `http://127.0.0.1:3000`.
- Runtime logs are written under `.margin/dev/logs/`.

Stop the local stack:

```bash
python scripts/dev.py stop
```

## Required Checks Before Tagging

Backend:

```bash
pip install -e ".[dev,data]"
ruff check src tests
pytest -q
```

Frontend:

```bash
cd web
npm ci
npm run lint
npm test
npm run build
cd ..
```

Docker configuration:

```bash
docker compose config --quiet
```

## Required CI Status

The default CI must pass without private provider secrets.

- Backend lint and tests use the same commands documented above.
- Backend CI imports `margin.api.main` and verifies key OpenAPI routes.
- Frontend install, lint, test, and build use the same commands documented above.
- Docker CI validates Compose configuration with `docker compose config --quiet`.
- Live provider calls are not required for default CI.

## Required Smoke-Test Evidence

Before tagging, record the date, command, exit code, and short result for:

- local startup path;
- backend checks;
- frontend checks;
- Docker Compose validation;
- one dashboard refresh using configured local providers, or a clear blocked
  status explaining missing provider budget/configuration.

Do not paste provider tokens, prompt bodies, or raw private data into release
notes.

## Known Non-Goals

Margin v0.1 is not a trading bot and does not manage brokerage accounts. It is a
local research assistant for candidate screening, evidence inspection, risk
review, and dashboard review.

The v0.1 release does not require:

- live-provider tests in default CI;
- Docker image builds in default CI;
- new strategy-performance claims;
- strategy-logic changes;
- contributor-governance documents.

## Release-Blocking Issues

Block the v0.1 tag if any of these are true:

- documented local startup path does not work on a clean checkout;
- default backend or frontend checks fail;
- CI requires private provider credentials;
- default tests call live Tushare, OpenAI, Tavily, web search, or similar services;
- Docker Compose configuration is invalid;
- README presents research metrics as future expectations;
- AI output is described as authoritative financial advice;
- generated logs, caches, local databases, or provider dumps are tracked.
