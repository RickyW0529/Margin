# 0202 成本与组合暴露计算 — 实施计划

## 1. 任务目标

实现 Portfolio Service 的成本与数量计算、组合风险引擎：单票仓位、行业集中度、风格暴露、相关性、流动性、波动率、回撤、事件集中风险。每日收盘后基于行情重算盈亏与风险。

## 2. 工作项拆解

- 0202.1 成本与数量计算 — 持仓成本、市值、盈亏（今日/累计）。
- 0202.2 组合风险引擎 — 单票仓位、行业集中度、风格暴露、相关性、流动性、波动率、回撤。
- 0202.3 事件集中风险 — 持仓即将发生的重要事件聚合。
- 0202.4 风险 API — GET /portfolios/{id}/risk。

## 3. 依赖关系

- 前置：0201（交易与持仓记录）。
- 被依赖：0203（基础面板展示）、0901（投资逻辑跟踪）、06（PortfolioTool）。
- 外部依赖：01-data_provider 行情（盈亏重算）。

## 4. 工时估算

- 0202.1：2 天
- 0202.2：3 天
- 0202.3：1 天
- 0202.4：1 天
- 合计：7 天。

## 5. 里程碑与交付物

- M1：成本与盈亏计算可用（第 2 天）。
- M2：组合风险引擎八维度齐全（第 5 天）。
- M3：事件集中风险聚合（第 6 天）。
- M4：风险 API 可用（第 7 天）。

## 6. 验收动作

- 持仓盈亏与组合风险指标计算正确；
- 行情缺失时沿用上一可用快照并标记 DATA_MISSING（对应 spec 02 §7）；
- 对应产品 §15 条目 7。
