# 0201 持仓录入（手工/CSV） — 实施计划

## 1. 任务目标

实现持仓与交易记录的手工录入、CSV/Excel 导入、券商导出文件适配插件接口。不默认连接券商账户、不保存券商密码、不自动下单。导入文件格式异常时拒绝写入并提示字段错误。

## 2. 工作项拆解

- 0201.1 手工交易录入接口 — POST /portfolios/{id}/trades。
- 0201.2 CSV/Excel 导入 — POST /portfolios/{id}/imports，字段校验与错误提示。
- 0201.3 券商导出文件适配插件协议 — BrokerImportPlugin 接口，MVP 提供示例适配。
- 0201.4 导入审计 — 记录录入时间、来源、原始行哈希。

## 3. 依赖关系

- 前置：0102（行情数据用于校验，对应 Gantt b1 after a2）。
- 被依赖：0202（成本与仓位计算）。
- 外部依赖：01-data_provider 证券元数据校验。

## 4. 工时估算

- 0201.1：2 天
- 0201.2：2 天
- 0201.3：2 天
- 0201.4：1 天
- 合计：7 天。

## 5. 里程碑与交付物

- M1：手工录入可用（第 2 天）。
- M2：CSV/Excel 导入含字段校验（第 4 天）。
- M3：券商适配插件协议与示例（第 6 天）。
- M4：导入审计完整（第 7 天）。

## 6. 验收动作

- 手工录入与 CSV 导入生成交易记录；
- 格式异常被拒绝并提示；
- 系统默认不执行真实交易、不保存券商密码（对应产品 §15 条目 10）。
