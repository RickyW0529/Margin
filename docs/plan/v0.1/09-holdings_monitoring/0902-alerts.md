# 0902 提醒引擎与盘中监控 — 实施计划

## 1. 任务目标

实现盘中监控（Price Poller + News Poller → 规则引擎 → 触发事件 → 证据检索 → 轻量 AI 解释 → 逻辑改变? → P0/P1 或 P2/P3 提醒）与提醒分级（P0 立即置顶/P1 交易时段/P2 面板+晚间汇总/P3 仅研究日志）。盘中不执行重新训练、全市场研究、任意 Agent 长链、自动下单或无规则限制的自由研究结论。

## 2. 工作项拆解

- 0902.1 Price/News Poller — 盘中价格与新闻轮询。
- 0902.2 规则引擎 — 确定性规则检测（价格触及失效阈值、行业暴露超限等）。
- 0902.3 证据检索与轻量 AI 解释 — 触发后 RAG 验证是否有新证据。
- 0902.4 提醒分级与通知 — P0–P3 分级，NotificationProvider 输出。

## 3. 依赖关系

- 前置：0901（投资逻辑状态）。
- 被依赖：0903（复盘记录）。
- 外部依赖：01 盘中价格、03 公告/新闻、05 证据检索。

## 4. 工时估算

- 0902.1：2 天
- 0902.2：2 天
- 0902.3：2 天
- 0902.4：1 天
- 合计：7 天。

## 5. 里程碑与交付物

- M1：Price/News Poller 可用（第 2 天）。
- M2：规则引擎九类提醒类型（第 4 天）。
- M3：证据检索与轻量解释（第 6 天）。
- M4：提醒分级与通知（第 7 天）。

## 6. 验收动作

- 价格触及失效阈值时触发提醒并经 RAG 验证（对应产品 §5.3）；
- 盘中不执行自动下单或任意 Agent 长链（对应 spec 09 §1）；
- 提醒按 P0–P3 分级输出。
