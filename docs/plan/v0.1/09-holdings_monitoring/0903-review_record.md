# 0903 复盘记录与操作历史 — 实施计划

## 1. 任务目标

实现复盘记录与操作历史视图：单个持仓详情含成本与盈亏、原始买入逻辑、当前证据、估值变化、模型排名变化、催化剂时间线、失效条件、操作历史。alerts API 查询。支持触发失效条件后处理时长度量（产品 §12.3 用户行为指标）。

## 2. 工作项拆解

- 0903.1 操作历史视图 — 交易、提醒、决策记录聚合。
- 0903.2 复盘记录 — 投资逻辑失效后的归因与复盘。
- 0903.3 alerts API — GET /positions/{id}/alerts。
- 0903.4 用户行为指标 — 触发失效条件后处理时长、研究信号与实际执行差异。

## 3. 依赖关系

- 前置：0902（提醒引擎）。
- 被依赖：无（持仓监控模块终点）。
- 外部依赖：02 交易记录、09 alert_event。

## 4. 工时估算

- 0903.1：2 天
- 0903.2：2 天
- 0903.3：1 天
- 0903.4：2 天
- 合计：7 天。

## 5. 里程碑与交付物

- M1：操作历史视图可用（第 2 天）。
- M2：复盘记录（第 4 天）。
- M3：alerts API 可用（第 5 天）。
- M4：用户行为指标度量（第 7 天）。

## 6. 验收动作

- 持仓详情可展开操作历史与失效条件（对应产品 §8.2）；
- 复盘记录完整保留（对应 spec 09 §8）；
- 触发失效条件后处理时长可度量。
